Comments: A single line comments uses // while a comment block will look like this /* */.
Having comments is a good practice in your code to have explanations for what certain snippets do and it is especially helpful if someone else is looking over your code. As you may know what everything does because you made it but someone else looking at your code may not.
The end of each JavaScript statement should have a semicolon to end it. JavaScript can place these automatically if you do not but this can lead to errors via the automation of placing semicolons in potentially the wrong location.
Why is there a need for Automatic Semicolon Insertion (ASI)?
A block does not need to be terminated by a semicolon.
